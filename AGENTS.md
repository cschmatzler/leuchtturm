# Workflow

- If asked a question, answer it. Do not edit files.
- Clarify ambiguous requirements before editing. Do not guess.
- Collaborate with the user; do not proceed silently.
- Never generate migrations. Tell the user to create them.
- When the user points out a mistake, fix it in the same turn.
- Never revert unrelated changes. They belong to the user.
- Never squash or amend commits. Create new commits.
- Do not edit autogenerated files, except when tooling updates them as part of the task.

# Style

- Prefer inline code over unnecessary helper functions.

# Vite+ Toolchain

```bash
vp lint         # Lint code
vp test         # Run tests
vp fmt          # Format code
vp check        # Lint + type check
```

- `vp lint` and `vp check` include type checking. Never run `tsc` directly.
- Import all testing/build utilities from `vite-plus`:

```typescript
import { describe, it, expect, vi } from "vite-plus/test";
import { defineConfig } from "vite-plus";
```

- After changes, always run `vp check` and `vp test`.
