# Workflow

- If asked a question, answer it. Do not continue editing files.
- Clarify ambiguous requirements before editing. Do not guess or make assumptions.
- Never generate migrations. Tell the user to create them.
- Never revert unrelated changes. They belong to the user.
- Never squash or amend commits. Create new commits.
- Use the Executor MCP wherever you can.
- Do not edit autogenerated files, except when tooling updates them as part of the task. This includes package manager lockfiles and route generation.
- When finishing a change, check the related and adjacent implementation to verify that your changes match the codebase style. For example, when editing a domain file in core, check two to three other core files. When editing an API handler, check two to three other API handlers. When you find a mismatch, fix it before committing and finishing the turn.

# Style

- Prefer inline code over unnecessary helper functions.
- Do not introduce constants for values that are only used to avoid repeating short literals or simple property accesses; inline them instead.

# Toolchain

```bash
aube run lint       # Lint code with Oxlint
aube run test       # Run tests with Vitest
aube run fmt        # Format code with Oxfmt
aube run check      # Check formatting with Oxfmt, then lint + type check with Oxlint
```

- `aube run lint` and the Oxlint part of `aube run check` include type checking. Never run `tsc` directly.
- Import test utilities from `vitest` and Vite utilities from `vite`:

```typescript
import { describe, it, expect, vi } from "vitest";
import { defineConfig } from "vite";
```

- After changes, always run `aube run check` and `aube run test`.
